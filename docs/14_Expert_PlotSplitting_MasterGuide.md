## PART 1 — THE BIG PICTURE: WHAT ARE YOU LOOKING AT?

### 1.1 What Is a Master Plan for an Industrial Estate?

A master plan for an industrial estate is a **top-down territorial blueprint** that divides a raw land area into functional zones. It answers five questions:

| Question | What It Means |
|---|---|
| **Where do factories go?** | Saleable industrial plots (J-series, C-series, A-series) |
| **How do trucks get around?** | Road network (primary + secondary roads) |
| **Where does water go?** | Retention ponds + wastewater treatment |
| **What keeps it legal/green?** | Green buffer zones along boundaries |
| **Where is the power?** | Main Electrical Substation location |

The entire goal is: **maximize saleable plot area while complying with regulations**.

## PART 3 — HOW PLOT SPLITTING ACTUALLY WORKS: THE EXPERT METHOD

### 3.1 The Four Layers of Plot Splitting Logic

```
Layer 1: REGULATORY (Non-negotiable)
         ↓
Layer 2: INFRASTRUCTURE (Roads, Utilities)
         ↓
Layer 3: MARKET DEMAND (What buyers want)
         ↓
Layer 4: GEOMETRY/AI OPTIMIZATION (How to fit it all)
```

---

### 3.2 LAYER 1 — Regulatory Constraints (Thailand Industrial Estate Authority — IEAT)

In Thailand, industrial estates must comply with **IEAT regulations**:

| Regulation | Requirement | Impact on Splitting |
|---|---|---|
| **Minimum Saleable Ratio** | ≥ 60% of total area | Can't over-allocate roads/green |
| **Green Buffer (Setback)** | 10–15m from boundary | Reduces usable edge land |
| **Road in Front of Each Plot** | Every plot must front a road | Determines minimum road network |
| **Water Retention** | ~5% of total area | Large ponds must be pre-planned |
| **Wastewater Treatment** | Mandatory facility | Pre-allocated zone required |
| **Utility Easements** | 6–10m along HV lines | Cannot build near power lines |

**In your constraints.json, this is encoded as:**
```json
{
  "boundary_setback_m": 15.0,
  "target_saleable_percentage": 70.0,
  "target_road_percentage": 20.0,
  "target_green_percentage": 5.0,
  "target_pond_percentage": 5.0
}
```

---

### 3.3 LAYER 2 — Road Network Logic (The Grid Backbone)

The road network is the skeleton. **Plots are what's left after roads are removed.**

#### Road Hierarchy in WHA Plans

| Road Type | Width | Function | In Your Plans |
|---|---|---|---|
| **Primary Road** | 24m | Main arterial (trucks) | The main N-S and E-W spines |
| **Secondary Road** | 16m | Between plot rows | Divides blocks into plots |
| **Service Road** | 10–12m | Back-of-plot access | Optional, rear loading |
| **Utility Corridor** | 6m | Pipes, cables | Along plot sides |

#### Why Roads Determine Plot Sizes

In Plan 5, you can see the perfect grid:
- Each row of plots is **bounded by roads on both long sides**
- Plot depth = distance between two parallel roads minus setbacks
- Plot width = interval between two cross-roads

**Formula:**
```
Plot Width  = Road Spacing (across) - 0 (roads don't subtract from width)
Plot Depth  = Road Spacing (along)  - 0 (setback internal only)
Plot Area   = Width × Depth
```

For the J-series standard plots:
- Typical depth: ~100m (between two secondary roads)
- Typical width: ~100–140m
- Result: ~10,000–14,000 m² = ~6–9 rai

#### The "Flexibility Rule" for Plot Splitting
Expert planners always design plots so they can be **merged or split later**:
- 2 × 7.5 rai plots can merge into 1 × 15 rai for a larger buyer
- 1 × 15 rai plot can split into 2 × 7.5 rai if demand is for smaller
- The road between them either keeps its **right-of-way** OR gets vacated in the title deed

### 3.5 LAYER 4 — The Geometry / AI Optimization

This is exactly what the WHA AI system (your codebase) is built to do.

#### The 5-Step Plot Splitting Algorithm

```
STEP 1: DEFINE THE DEVELOPABLE ENVELOPE
        Total site area
        MINUS boundary setback (15m buffer)
        MINUS road reservations
        MINUS pond/utility zones
        = Net Developable Polygon

STEP 2: ESTABLISH THE ROAD GRID
        Place primary roads (24m wide)
        Place secondary roads (16m wide)
        This creates rectangular "superblocks"

STEP 3: SUBDIVIDE SUPERBLOCKS INTO PLOTS
        For each superblock:
          target_width = 100–150m
          target_depth = 100–120m
          if superblock_area > max_plot_size → split
          if superblock_area < min_plot_size → merge with neighbor

STEP 4: HANDLE IRREGULAR GEOMETRY
        Topographic constraints (contour lines visible in Plans 1, 4, 5)
        Corner plots get special treatment (triangular cutoffs)
        Edge plots near pond get curved/angled boundaries

STEP 5: VALIDATE AND SCORE
        Check: saleable% ≥ 70%
        Check: each plot has road frontage
        Check: no plot < 5,000 m² or > 15,000 m² (from constraints.json)
        Score: maximize total saleable area
```

## PART 6 — TERRAIN & TOPOGRAPHY: WHY THE SHAPES ARE NOT PERFECT SQUARES

### 6.1 What the Contour Lines Tell You

In Plans 1, 4, and 5, you can see **contour lines** (the curved lines in the background). These are elevation lines. In industrial estate planning:

- **Each contour = 1–2 meter elevation change**
- The site has significant **terrain variation** (this is "rugged terrain" per the Engineer's Note in Plan 1)
- This is why the plan says: *"Extensive earthwork and cut-and-fill operations for industrial pads in rugged terrain"*

### 6.2 Why This Affects Plot Shapes

**Flat plots are mandatory for factories.** Every plot must be leveled. The constraints:
- Steep areas → smaller plots (less earthwork volume per plot)
- Valley areas → used for retention ponds (natural low point)
- Ridge areas → first to be developed (easiest earthwork)

**In Plan 2 (GVTP) and Plans 5/6 (RY36):** The **A-series plots** on the right side follow irregular shapes **because they're following natural topography** — they're the last areas to develop where straight rectangles don't work.

## PART 8 — THE COMPLETE PLOT SPLITTING DECISION TREE

This is the expert mental model for deciding how to split any plot:

```
START: Raw land area
       │
       ▼
Is area > max_plot_size (15,000 m²)?
       │YES                    │NO
       ▼                       ▼
Split into 2 equal parts   Is area > min_plot_size (5,000 m²)?
       │                       │YES            │NO
       ▼                       ▼               ▼
Re-evaluate each half      KEEP as 1 plot    MERGE with
       │                                     adjacent plot
       ▼
Does each half have road frontage?
       │YES                │NO
       ▼                   ▼
Both become              One half needs a
separate plots           service road added
                         OR merge back

THEN FOR IRREGULAR SHAPES:
Does plot boundary follow topography?
       │YES                │NO
       ▼                   ▼
Accept irregular shape   Force rectangular
(A-series style)         (J-series style — preferred)
```
